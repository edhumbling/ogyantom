For all UI design, implementation, redesign, and review work in this project, apply Vercel Web Interface Guidelines from https://vercel.com/design/guidelines.

Before changing interface code, check accessibility, keyboard support, visible focus, hit targets, motion preferences, responsive layout, overflow, form semantics, loading/error/empty states, image sizing, and clear navigation.

Prefer native semantics (`button`, `a`/`Link`, `label`, headings) before ARIA. Icon-only controls need accessible names. Interactive states must increase contrast. Avoid `transition: all`, disabled browser zoom, paste blocking, unlabeled form controls, unwanted scrollbars, and dead-end screens.

Keep the existing Ogya Ntom visual language unless the user asks for a new direction.
